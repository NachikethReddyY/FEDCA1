Introduction

This study guide is designed to help you review the concepts covered in "Topic 02 Digital Representation." It includes a quiz, essay questions, and a glossary of key terms. Understanding how information is represented digitally is fundamental to computer science. From simple numbers to complex images and sounds, everything inside a computer is ultimately stored and processed as binary data.

Quiz Answer Key

The most natural way to represent a number in a computer system is by a string of bits, called a binary number. The denary (or decimal) number system is the number system people use daily. It has a radix of 10, meaning it uses 10 digits (0 through 9). The weighted sum method for binary to decimal conversion involves multiplying each binary digit by its corresponding positional value (powers of 2) and summing the results. To convert a decimal number to binary, you repeatedly divide the decimal number by 2, keeping track of the remainders. The binary representation is read from the last remainder (MSB) to the first (LSB). The range of numbers that can be represented by an unsigned binary number with N bits is from $0$ to $2^N-1$. Octal (Base 8) and Hexadecimal (Base 16) systems are used as a shorthand to represent binary numbers because they can more compactly represent strings of bits, making them easier for humans to read and write accurately. In signed integer representation, the leftmost bit (most significant bit, or MSB) is used to indicate the sign: $0$ for positive and $1$ for negative. In 1's complement, negative values are obtained by complementing each bit of the corresponding positive number. In 2's complement, you form the 1's complement and then add $1$. The four rules for adding binary numbers are: $0+0=0$, $0+1=1$, $1+1=10$ (binary for 2), and $1+1+1=11$ (binary for 3). Overflow in signed integer arithmetic occurs when the result of an arithmetic operation does not fit within the representable number range for the given number of bits.

Number Representation

Binary Numbers

The most natural way to represent a number in a computer system is by a string of bits, called a binary number. A bit is a single binary digit ($0$ or $1$).

Denary (Decimal) Number System

People use the denary (or decimal) number system in their day-to-day lives. This system has a radix (base) of $10$, using digits $0$ through $9$. Each digit's value depends on its position, known as its positional value (powers of $10$).

Converting Binary to Decimal (Weighted Sum Method)

To convert a binary number to decimal, use the weighted sum method. Multiply each binary digit by its corresponding positional value (powers of $2$) and sum the results. The rightmost bit is the Least Significant Bit (LSB), and the leftmost is the Most Significant Bit (MSB).

Unsigned Integers

Unsigned integers represent only non-negative values. If there are $N$ bits in the binary number, the range of the number is from $0$ to $2^N-1$. For example, with $3$ bits, the range is $0$ to $2^3-1 = 7$ ($000$ to $111$). For $8$ bits, the range is $0$ to $2^8-1 = 255$. Unsigned numbers are often used for memory addresses, cluster numbers in file systems, and process identifiers (PIDs).

Octal (Base 8) and Hexadecimal (Base 16)

Strings of bits can be difficult for humans to read and write accurately. Octal (Base $8$) and Hexadecimal (Base $16$) systems are used as a shorthand to represent binary numbers, making them more compact and readable. Hexadecimal uses digits $0-9$ and letters $A-F$ (where $A=10, B=11, \dots, F=15$). Each hexadecimal digit corresponds to exactly $4$ binary bits.

Denoting Hexadecimal:

  • Place $0x$ before the hex numeral: e.g., $0x3573$
  • Append a lowercase letter $h$ after the numeral: e.g., $3573h$
  • Use the $16$ subscript: e.g., $3573_{16}$
  • Specify base $16$: e.g., $3573$ (base $16$)

Signed Integers

Signed integers represent both positive and negative values. The leftmost bit (MSB) indicates the sign: $0$ for positive and $1$ for negative. There are three primary ways to represent signed integers:

  • Sign-and-Magnitude: The MSB indicates the sign, and the remaining bits represent the magnitude.
  • 1's Complement: Negative values are obtained by inverting all bits of the corresponding positive number.
  • 2's Complement: Negative values are obtained by taking the 1's complement and then adding $1$. This is the most efficient and commonly used method in modern computers for arithmetic operations.

Example of 2's Complement: To form $-5$ from $+5$ ($0101$ in $4$-bit):

  • Original: $0101$ ($+5$)
  • 1's Complement (invert bits): $1010$
  • Add $1$: $1010 + 1 = 1011$ ($-5$)

Arithmetic Operations

Binary Addition

The four rules for adding binary numbers are:

  • $0 + 0 = 0$
  • $1 + 0 = 1$
  • $1 + 1 = 10$ (binary for $2$, carry $1$)
  • $1 + 1 + 1 = 11$ (binary for $3$, carry $1$)

Binary Subtraction

Subtraction is typically performed by forming the 2's complement of the subtrahend (the number being subtracted) and then performing normal binary addition.

Example: Subtract $-5$ ($1011$) from $-2$ ($1110$)

  • Original: $1110$ ($-2$) $- 1011$ ($-5$)
  • 2's complement of $1011$ ($-5$) is $0101$ ($+5$)
  • Perform addition: $1110 + 0101 = 0011$ ($-2 + 5 = +3$)

Overflow

Overflow occurs when the result of an arithmetic operation does not fit within the representable number range for the given number of bits. For $N$-bit signed integers, the range is from $-2^{N-1}$ to $+2^{N-1}-1$. For example, with $4$ bits, the range is $-8$ to $+7$.

Conditions for Overflow:

  • Addition: If two numbers with the same sign are added, and the result has the opposite sign.
    • $(+A) + (+B) = -C$
    • $(-A) + (-B) = +C$
  • Subtraction: If two numbers with different signs are subtracted, and the result has the same sign as the subtrahend (the number being subtracted).
    • $(+A) - (-B) = -C$
    • $(-A) - (+B) = +C$

Case Study: Ariane 5 Rocket Crash
A famous example of the consequences of overflow is the Ariane 5 rocket explosion. A small computer program tried to stuff a $64$-bit number into a $16$-bit space, leading to an arithmetic error and total destruction. This highlights the critical importance of handling number ranges and potential overflows in software development.

Sign Extension

Sign extension is the process of increasing the number of bits of a signed binary number while preserving its sign and value. This is done by replicating the sign bit (MSB) to the left.

Character Representation

ASCII (American Standard Code for Information Interchange)

ASCII is a character encoding standard that uses $7$-bit codes to represent text characters. This allows for $128$ unique characters, sufficient for English alphabet, numbers, and basic symbols.

Unicode

While ASCII is sufficient for English, it lacks support for accents in European languages and larger alphabets like Cyrillic or Chinese Mandarin. Unicode was created to address this limitation, providing a standard to represent characters from virtually all languages. This allows computers to be used by people worldwide regardless of their native language.

Representing Sound

Sound needs to be converted into binary for computers to process it. This involves capturing sound (e.g., via a microphone) and converting it into a digital signal through sampling. The samples are then converted to binary, typically recorded to the nearest whole number.

Encoding Images

When an image is encoded, it is broken up into individual pixels. Each pixel is the smallest controllable element of a picture on the screen. Color is typically represented using a combination of Red, Green, and Blue (RGB) values, where each color component is assigned a numerical value (e.g., $0-255$).

Glossary of Key Terms

Binary Number:
A number represented using only the digits $0$ and $1$.
Bit:
A single binary digit ($0$ or $1$).
Denary Number (Decimal Number):
The base-$10$ number system commonly used by humans.
Radix (Base):
The number of unique digits, including zero, used to represent numbers in a positional numeral system.
Positional Value:
The value assigned to a digit based on its position in a number. In decimal, these are powers of $10$; in binary, they are powers of $2$.
Weighted Sum Method:
A method for converting a number from a different base to decimal by multiplying each digit by its positional value and summing the results.
LSB (Least Significant Bit):
The rightmost bit in a binary number, representing the smallest positional value.
MSB (Most Significant Bit):
The leftmost bit in a binary number, representing the largest positional value.
Unsigned Integers:
Binary numbers that represent only non-negative values.
Signed Integers:
Binary numbers that represent both positive and negative values.
Sign-and-Magnitude:
A method of representing signed integers where the most significant bit indicates the sign and the remaining bits represent the magnitude of the number.
1's Complement:
A method of representing negative signed integers by inverting all the bits of the corresponding positive number.
2's Complement:
A method of representing negative signed integers by taking the 1's complement and adding $1$.
Octal (Base 8):
A number system with a radix of $8$, using digits $0-7$. Often used as a shorthand for binary.
Hexadecimal (Base 16):
A number system with a radix of $16$, using digits $0-9$ and letters $A-F$. Often used as a shorthand for binary.
ASCII (American Standard Code for Information Interchange):
A character encoding standard that uses $7$-bit codes to represent text characters.
Unicode:
A character encoding standard designed to represent characters from all languages.
Pixel:
The smallest controllable element of a picture represented on the screen.
Overflow:
A condition that occurs in arithmetic operations when the calculated result is larger in magnitude than the maximum value that can be represented with the given number of bits.
Sign Extension:
The process of increasing the number of bits of a signed binary number while preserving its sign and value.
