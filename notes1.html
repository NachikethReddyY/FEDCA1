Computer Architecture Notes

Overview of Computer Concepts

Most devices encountered when working with a network involve a computer. This includes workstations (PCs and laptops), network servers running operating systems like Windows, Linux, UNIX, and Mac OS, as well as specialized computers like routers and switches used to move data.

Basic Functions of a Computer

A computer’s functions can be broken down into three basic tasks:

  • Input: A user types the letter ‘A’ on the keyboard, which results in sending a code representing the letter ‘A’ to the computer.
  • Processing: The computer’s CPU determines what letter was typed by looking up the keyboard code in a table.
  • Output: The CPU sends instructions to the graphics cards to display the letter ‘A’, which is then sent to the computer monitor.

Components of a Computer

A computer system is comprised of several key components working together.

Input Components

Common user-controlled input devices include keyboards, microphones, Webcams, and scanners. External interfaces like serial, FireWire, and USB ports are also used to get input from external devices. Storage devices such as hard disks and CDs/DVDs can also serve as input sources.

Processing Components

The CPU (Central Processing Unit) is a computer’s main processing component. It executes instructions from computer programs and the operating system. Modern CPUs often have multiple processors called cores (multicore CPUs), enabling simultaneous instruction execution and better overall performance.

Output Components

The most obvious output devices are monitors and printers. Other output components include storage devices, network cards, and speakers. External interfaces, such as a USB port, can also function as output (e.g., writing files to a disk).

Storage Components

The amount of storage a computer has significantly impacts performance. Most storage components act as both input and output devices. Storage is categorized into short-term and long-term storage.

RAM: Short-Term Storage

Random Access Memory (RAM) is volatile, meaning its contents are lost when power is turned off. It's referred to as "working storage" and is crucial for efficient computer operation. Insufficient RAM can lead to the computer using disk space as virtual memory.

Virtual Memory

Part of disk storage can be designated as virtual memory. The CPU can only access data/code directly in RAM. Less frequently used data/code are moved to virtual memory, and then moved back to RAM when needed by the CPU.

Long-Term Storage

Long-term storage is non-volatile, retaining data even without power. Examples include hard disks, CDs/DVDs, and USB flash drives. These are used to store documents, multimedia files, applications, and OS files. The required storage amount depends on the type and quantity of files.

Personal Computer Hardware

The four major PC components are:

  • Motherboard
  • Hard drive
  • RAM
  • BIOS/CMOS

The Motherboard and Its Components

The motherboard is the main printed circuit board in general-purpose computers and other expandable systems. It holds and allows communication between many of the crucial electronic components of a system, such as the central processing unit (CPU) and memory, and provides connectors for other peripherals.

Key Components of a Motherboard

Component Description
CPU socket The CPU is installed in this socket.
PCI bus expansion slots Used to add functionality to a PC by adding expansion cards that have a Peripheral Component Interconnect (PCI) connector.
PCI-Express expansion slots PCI-Express supersedes PCI and supports faster data transfer speeds. The larger slots are suitable for high-performance expansion cards, such as graphics cards and disk controllers. The smaller slots are best suited to sound cards and network interface cards.
RAM slots Slots for installing RAM on the motherboard.
Chipset with heat sinks The chipset consists of two chips referred to as the Northbridge and the Southbridge. These chips control data transfers between memory, expansion slots, I/O devices, and the CPU. The heat sink sits on top of the chipset to prevent it from overheating.
SATA connectors Used for connecting hard drives and CD/DVD drives that use the Serial AT Attachment (SATA) specification.
IDE connector Used for connecting Integrated Drive Electronics (IDE) hard drives and CD/DVD-ROM drives. Most systems now use SATA for hard drives and IDE for CD/DVD-ROM drives.
Main power connector This connector is where the motherboard receives power from the system power supply.

Computer Bus Fundamentals

A bus is a collection of wires carrying signals (data, address, or control) from one place to another on the computer. All data entering or leaving a computer passes through the motherboard. Buses exist between the CPU and RAM, CPU and disk drives, and CPU and expansion slots.

Types of Buses:

  • Data Bus: Carries data signals from main memory (RAM) to CPU and vice versa, or from main memory (RAM) to input/output (I/O) devices and vice versa.
  • Address Bus: Carries address signals, such as a memory location or port/interface where an input or output device is attached.
  • Control Bus: Carries control signals, such as read or write commands, from the CPU to memory or a port/interface.

The data bus carries the data to be stored, while the address bus carries the location where it should be stored.

I/O Polling and Interrupt

External devices like a mouse, keyboard, scanner, and printer require CPU attention. When an event occurs (e.g., a mouse click), the CPU needs to respond. Interrupt and Polling are two methods for handling events generated by I/O devices.

  • Polling: The CPU continuously checks I/O devices at regular intervals to see if they need CPU service.
  • Interrupt: The I/O device interrupts the CPU to signal that it requires CPU service.

Both methods allow the CPU to stop its current task and respond to a more important one.

Hard Drive Fundamentals

A hard drive is the primary long-term storage component. It consists of magnetic disks called "platters" that store data as magnetic pulses. Hard drives store documents, applications, and the operating system.

Solid State Drives (SSDs)

SSDs are increasingly used in place of hard drives due to their speed and reliability. They use flash memory, have no moving parts, and offer faster access times. While more expensive, they are common in mobile devices and high-performance desktops/servers.

BIOS/CMOS Fundamentals

The BIOS (Basic Input/Output System) is a set of instructions on a motherboard chip that tells the CPU to perform tasks when the computer powers on, including a Power-On Self Test (POST). Hardware configurations are stored in CMOS (Complementary Metal-Oxide Semiconductor) memory.

Computer Boot Procedure

  1. Power is applied to the motherboard.
  2. The CPU starts.
  3. The CPU carries out the BIOS startup routines, including the POST.
  4. Boot devices, as specified in the BIOS configuration, are searched for an OS.
  5. The OS is loaded into RAM.
  6. OS services are started.

The Fundamentals of Network Communication

A computer network consists of two or more computers connected by a transmission medium (e.g., cable or air waves). To access the Internet, a computer must connect to a network.

NIC Basics

Attaching a computer to a network requires a Network Interface Card (NIC). The NIC creates and mediates the connection between the computer and the networking medium (copper wire, fiber-optic cable, or airwaves). Each NIC has a unique MAC address (Media Access Control Address).

Wireless NICs

Wireless NICs must be chosen based on the type of wireless Access Point (AP) being used (e.g., Wireless-n, 802.11ac, 802.11 a/b/g/n). The letters refer to the wireless networking standard supported. Wireless NICs connect to a network using a Service Set Identifier (SSID), which is the name assigned to the wireless network. A security key or username/password may also be required.

Instruction Cycle

Even though computer systems run complex programs, their basic operation is simple, relying on repeating basic operations billions of times per second. A single cycle of operation, known as the instruction cycle or machine cycle, consists of several phases: Fetch, Decode, Execute, Memory Access, and Write Back.

Step Action
1 Fetch an instruction and increment the program counter.
2 Decode the instruction and read registers from the register file.
3 Perform an ALU operation.
4 Read or write memory data if the instruction involves a memory operand.
5 Write the result into the destination register, if needed.

Instruction Types

There are three basic instruction types:

  • Load: Read a data operand from memory or an input device into the processor.
  • Store: Write a data operand from a processor register to memory or an output device.
  • Operate: Perform an arithmetic or logic operation on data operands in processor registers.

Example Program: $C = A + B$

Load    R2, A
Load    R3, B
Add     R4, R2, R3
Store   R4, C

Here, A, B, and C are memory word addresses, and Ri are processor registers.

Processor Components (Instruction Cycle)

  • The Program Counter (PC) register holds the memory address of the next instruction to be fetched and executed by the CPU.
  • The Instruction Register (IR) holds the current instruction.
  • General-purpose registers hold data and addresses.
  • Control circuits and the Arithmetic and Logic Unit (ALU) fetch and execute instructions.

Fetching and Executing Instructions

Example: Load R2, LOC

The processor control circuits perform the following steps:

  1. Send address in PC to memory; issue Read.
  2. Load instruction from memory into IR.
  3. Increment PC to point to next instruction.
  4. Send address LOC to memory; issue Read.
  5. Load word from memory into register R2.

Program Counter (PC)

A program counter is a register in a computer processor that contains the address (location) of the instruction being executed at the current time. As each instruction is fetched, the program counter increases its stored value by 1. After each instruction is fetched, the program counter points to the next instruction in the sequence. When the computer restarts or is reset, the program counter normally reverts to 0.

Handling I/O Devices

An application program can:

  • Read data (such as a keyboard character) from an input device.
  • Write data (such as a letter character) to an output display screen.
  • Sense the readiness of an input or output (I/O) device to perform a transfer.

Performance

How quickly can a program be executed? Some factors influencing performance include:

  • Speed of electronic circuits in the processor.
  • Access times to the cache and main memory.
  • Design of the instruction set.
  • Number of operations that can be done at the same time (parallelism).

Technology

Improvements in transistor materials lead to faster instruction execution. Advances in process technology allow more transistors on a chip, resulting in more logic functionality and memory storage capacity.

Performance - Parallelism

Multicore processors enable multiple processing units to be fabricated on a single chip. Each unit is called a "core," and the complete chip is referred to as a "processor" (e.g., dual-core, quad-core, octo-core processors).

Little Man Computer (LMC)

The Little Man Computer (LMC) is an instructional model of a computer, created by Dr. Stuart Madnick in 1965. It is used to teach students the basic architecture and operation of a computer, particularly the Fetch-Decode-Execute cycle. The LMC simplifies the complex components of a real computer into an easily understandable analogy.

Key Components of LMC:

  • Little Man: Represents the CPU (Central Processing Unit). He performs all operations.
  • Mailboxes (Memory): 100 mailboxes (numbered 00-99), each capable of holding a 3-digit instruction or data value. This represents the computer's main memory.
  • Accumulator: A temporary storage area where the Little Man performs arithmetic operations. All calculations are done here.
  • Program Counter (PC) / Location Counter: Holds the address of the next instruction to be executed. The Little Man looks at this to know which mailbox to visit next.
  • Instruction Register (IR): Holds the instruction currently being executed.
  • Input/Output (I/O) Baskets: Represent input and output devices. The Little Man can read numbers from the In-Basket and write numbers to the Out-Basket.

LMC Instruction Cycle and "Next Step" Phrasing:

When describing the "next step" in the Little Man Computer, you are essentially detailing the phases of the Fetch-Decode-Execute cycle for a given instruction. The phrasing should clearly state what the Little Man (CPU) does, what registers are affected, and what memory locations are accessed.

Let's consider an example instruction: 5XX (LOAD data from mailbox XX into the Accumulator).

Example Phrasing for "Next Step" (assuming PC is currently at address 05, and instruction 520 is in mailbox 05):

  1. Fetch Phase:

    "The Little Man looks at the Program Counter (PC), which currently holds 05. He then goes to mailbox 05, fetches the instruction 520, and places it into the Instruction Register (IR). The Program Counter is then incremented to 06, pointing to the next instruction."

  2. Decode Phase:

    "The Little Man decodes the instruction 520 from the Instruction Register. He recognizes the opcode 5 as a 'LOAD' operation, meaning he needs to retrieve data from the memory address specified by the operand 20."

  3. Execute Phase:

    "The Little Man goes to mailbox 20, reads the data value stored there (e.g., if mailbox 20 contains 123), and places this value (123) into the Accumulator, overwriting any previous value. The instruction is now complete."

For other instructions, the execution phase will differ based on the opcode. For instance, an ADD instruction would involve fetching data from a mailbox and adding it to the current value in the Accumulator. A STORE instruction would involve taking the value from the Accumulator and writing it to a specified mailbox.

The key is to always explicitly state:

  • What the Little Man (CPU) is doing.
  • Which registers (PC, IR, Accumulator) are involved and what their values become.
  • Which mailboxes (memory locations) are accessed and for what purpose (read/write).
